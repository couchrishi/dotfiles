The Problem

Every new machine is a blank slate

  • Reinstalling tools, reconfiguring shells, re-setting permissions
  • Claude Code needs Vertex AI env vars, model pinning, hooks, MCP servers
  • No guardrails: AI can push to main, read .env files, run rm -rf
  • Formatters, linters, git aliases all need setup from scratch
  • GCP Cloud Workstations are ephemeral — config vanishes

05 / 16
Vertex AI

No API Keys. Just gcloud.

.bash_claude
# Use Vertex AI backend (ADC auth)
export CLAUDE_CODE_USE_VERTEX=1
export CLOUD_ML_REGION=global
# Pin model versions for stability
export ANTHROPIC_MODEL="claude-opus-4-6"
export ANTHROPIC_DEFAULT_SONNET_MODEL="claude-sonnet-4-6"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="claude-haiku-4-5@20251001"

Model pinning prevents breakage when new versions ship on Vertex. No API keys ever touch disk — gcloud ADC handles auth.

06 / 16
Permissions

Allow What's Needed. Deny What's Dangerous.

Allow

  • File ops: Read, Write, Edit, Glob, Grep
  • Dev tools: git, npm, python, go, make
  • Cloud: gcloud, docker, terraform
  • Utilities: jq, curl, grep, sort

Deny

  • Read .env, credentials, secrets, .pem
  • Read id_rsa (SSH keys)
  • Write to .env or credentials
  • rm -rf, sudo, chmod 777

Defined in settings.json — enforced at the tool level before commands run

07 / 16
Hooks

Automated Guardrails

PreToolUse: Block Push to Main
Intercepts every Bash command. If it matches git push.*main|master, it exits with an error and tells Claude to use a feature branch.
PostToolUse: Auto-Format
After every file edit, runs the right formatter: prettier for JS/TS/CSS, black for Python, gofmt for Go, terraform fmt for TF.

Hooks are deterministic and 100% enforced — unlike CLAUDE.md instructions, they can't be ignored

AI Instructions

CLAUDE.md — The System Prompt

Loaded every session. Short, opinionated, non-negotiable.

Behavior
Be direct. Plan before coding. Prefer simple solutions. Admit uncertainty.
Code Style
Explicit over clever. Meaningful names. Handle errors — no silent failures.
Git & GitHub
Use gh CLI. Conventional commits. Never commit to main.
GCP / Cloud
Use gcloud CLI. Never hardcode secrets. Use Secret Manager.
